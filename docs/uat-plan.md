# UAT計画

## 目的

MVPのWeb/PWAツールが、英語しかできない初訪日の観光客にとって、実際の移動前・移動中に使える状態かを受け入れ判定する。

このUATでは、実装品質だけでなく、利用者が次の行動を迷わず選べるか、人に見せる日本語が現場で使えるか、Webコンテンツ・収益導線が移動中の邪魔になっていないかを確認する。

## UATの位置づけ

| ドキュメント | 役割 |
|---|---|
| `scenario-test.md` | 仕様ドキュメント上のシナリオ検証 |
| `manual-test-checklist.md` | 実装後の画面操作チェック |
| `uat-plan.md` | リリース可否を判断する受け入れテスト |
| `uat-execution-report.md` | UAT実行結果と残る確認事項 |

## 対象範囲

UAT対象:
- 英語Webサイト + スマホ対応Webツール
- PWA化した場合のホーム画面起動、最低限のオフライン表示
- 主要交通手段の利用前準備
- 駅・バス停到着後から乗車直前までの確認
- Google Mapsへ移動し、戻った後の確認
- 人に見せる日本語表示
- タクシー切り替え
- Webコンテンツ、アフィリエイト、収益導線の表示位置

UAT対象外:
- ネイティブアプリ固有機能
- 乗車中・降車後の詳細支援
- 券売機の詳細操作支援、券売機撮影補助
- その他交通手段の会社・施設別ルール
- SEO記事やキーワード設計

## 想定テスター

必須:
- 英語UIだけで操作する初訪日観光客役
- 日本の交通に詳しくない人
- スマホだけで操作する人

可能なら追加:
- 日本語が少し分かるが交通に不慣れな人
- 荷物が多い旅行者役
- 子連れ・高齢者・疲労状態を想定するテスター
- 日本側確認相手役: 駅員、バス運転手、ホテルスタッフ、周囲の人

## テスト環境

端末:
- iPhone Safari
- Android Chrome
- 画面幅が狭いスマホ
- 通信が不安定な状態を再現できる端末

利用条件:
- 片手操作
- 屋外の明るい場所
- 駅・バス停のように短時間で判断が必要な状況
- Google Mapsを開いてから本ツールへ戻る操作

## 判定基準

| 判定 | 意味 |
|---|---|
| PASS | 迷わず進める。文言・ボタン・復帰導線が十分 |
| PASS with Notes | 進めるが、文言、順序、表示量に改善余地あり |
| FAIL | 次の行動が分からない、戻れない、現場で使えない |
| BLOCKED | 実装未完了、外部サービス不具合、環境問題で確認不能 |

リリース判定:
- CriticalケースはすべてPASS。
- HighケースはFAILなし。
- MediumケースはFAILがあっても、回避導線または後続対応方針が明確。
- 人に見せる日本語画面、乗車直前画面、緊急時の短い行動画面に収益導線が出ていない。

## 受け入れ基準

共通:
- 英語だけで次の操作が分かる。
- 日本語は人に見せる文として大きく表示される。
- 日本語文には本人確認用の英語訳がある。
- 1画面1目的になっている。
- 主ボタンは原則3つ程度に収まっている。
- 長い説明は補足に回っている。
- `Open Google Maps` 後に戻った時、次に何を確認するか分かる。
- 迷った時に `Show destination in JP`、`Open Google Maps`、`Switch to taxi` のいずれかへ進める。
- 夜間・地方・本数が少ない条件では警告が見える。
- 収益導線は、旅行前、Google Maps復帰後、警告からの復帰導線、到着後、荷物が多い時に限定される。

不合格にする条件:
- 日本語を読めない利用者が、次の操作を英語で判断できない。
- 人に見せる日本語文が長すぎる、または「確認したいです」に寄りすぎて目的がすぐ伝わらない。
- 人に見せる画面へ進むボタンが `Show Japanese` のような内部名で、誰に何を見せるのか分からない。
- バス利用時に選択肢が多すぎて止まる。
- バス停名だけ確認し、方向確認へ進めない。
- Google Mapsから戻った後にFlowへ戻れない。
- 夜間・本数少ない地域でタクシー切り替えが見つからない。
- 乗車直前や運転手向け画面に広告・アフィリエイト導線が出る。

| ID | 優先度 | ケース | 合格条件 |
|---|---|---|---|
| UAT-01 | Critical | 初回利用から目的地設定 | 英語だけで最終目的地と次に行く場所を設定できる |
| UAT-01-A | Critical | 目的地名と地域ヒント入力 | Google Mapsで調べた場所名と地域ヒントを入力できる |
| UAT-02 | Critical | Google Mapsを開いて戻る | 戻った後、次の確認画面へ迷わず進める |
| UAT-02-A | Critical | Google Maps上の目的地確認 | 表示された場所が意図した目的地か確認し、違う場合は修正へ戻れる |
| UAT-03 | Critical | 人に見せる日本語 | 日本語1文と英語訳が表示され、原則「〇〇したいです」で目的が短く伝わり、誰に見せるか分かる |
| UAT-04 | Critical | 路線バス準備 | 現金・小銭、降車停留所、方向確認、乗車直前確認へ進める |
| UAT-05 | Critical | バス停は正しいが方向が違う | 方向と降車停留所を一緒に確認でき、違う場合は戻れる |
| UAT-06 | Critical | バスが満員で乗れない | 次の同系統待ちとタクシー切り替えを選べる。混雑理由が短く分かる |
| UAT-07 | Critical | 夜間・本数が少ない地域 | 次の便・最終便確認とタクシー切り替えが見える |
| UAT-08 | High | 徒歩準備 | Google Maps徒歩ルートと人に見せる文へ進める |
| UAT-09 | High | タクシー準備 | 最終目的地まで行くか、途中地点まで行くかを間違えず選べる |
| UAT-10 | High | タクシー表示先選択 | ホテルスタッフ向けとドライバー向けが混ざらない |
| UAT-11 | High | 普通電車・地下鉄・モノレール | 駅、改札、ホーム、方向、停車駅を確認できる |
| UAT-12 | High | 新幹線・特急 自由席 | 乗車券、特急券、自由席車両、乗車位置を確認できる |
| UAT-13 | High | 新幹線・特急 指定席 | 列車名、号数、号車、座席、荷物を確認できる |
| UAT-14 | High | 高速バス・空港バス・夜行バス | 予約、乗車券、便名、乗り場、集合時刻、荷物を確認できる |
| UAT-15 | Medium | 路面電車 | バス種別選択から路線バス簡易フローへ進める |
| UAT-16 | Medium | その他の移動手段 | 詳細Flowに入らず、Google Maps、窓口、係員、公式案内へ渡せる |
| UAT-17 | Medium | 荷物が多い | タクシー、荷物預かり、空港送迎などの補助導線が邪魔にならず出る |
| UAT-18 | Medium | 到着後・最後の徒歩 | 入口、受付、集合場所を確認できる |
| UAT-19 | Medium | 通信不安定 | 保存済みの最低情報と人に見せる文を確認できる |
| UAT-20 | Medium | 収益導線 | 表示場所が自然で、移動中の主判断を邪魔しない |

## UAT実施手順

1. テスターに目的地と出発状況だけを渡す。
2. テスターは英語UIだけで操作する。
3. Google Mapsを開く場面では、実際に外部遷移して戻る。
4. 人に聞く場面では、日本側確認相手役に画面を見せる。
5. テスターは迷った箇所、読まなかった説明、押したくなかったボタンを記録する。
6. 各ケースをPASS / PASS with Notes / FAIL / BLOCKEDで判定する。
7. FAILは、文言修正、画面順序修正、ボタン削減、補足移動、後続対応のいずれかに分類する。
